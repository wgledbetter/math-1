1 The <b>Stan Math Library</b> is a C++, reverse-mode automatic
2 differentiation library designed to be usable, extensive and
3 extensible, efficient, scalable, stable, portable, and redistributable
4 in order to facilitate the construction and utilization of algorithms
5 that utilize derivatives.

41 Installation
42 ------------
43 The Stan Math Library is a C++ library which depends on the Intel TBB
44 library and requires for some functionality (ordinary differential
45 equations and root solving) the Sundials library. The build system is
46 the make facility, which is used to manage all dependencies.
47 
48 A simple hello world program using Stan Math is as follows:
49 
50 ```cpp
51 #include <stan/math.hpp>
52 #include <iostream>
53 
54 int main() {
55  std::cout << "log normal(1 | 2, 3)="
56  << stan::math::normal_log(1, 2, 3)
57  << std::endl;
58 }
59 ```
60 
61 If this is in the file `/path/to/foo/foo.cpp`, then you can compile
62 and run this with something like this, with the `/path/to` business
63 replaced with actual paths:
64 
65 ```bash
66 > cd /path/to/foo
67 > make -j4 -f /path/to/stan-math/make/standalone math-libs
68 > make -f /path/to/stan-math/make/standalone foo
69 > ./foo
70 log normal(1 | 2, 3)=-2.07311
71 ```
72 
73 The first make command with the `math-libs` target ensures that all
74 binary dependencies of Stan Math are built and ready to use. The `-j4`
75 instructs `make` to use 4 cores concurrently which should be adapted
76 to your needs. The second make command ensures that the Stan Math
77 sources and all of the dependencies are available to the compiler when
78 building `foo`.
79 
80 An example of a real instantiation whenever the path to Stan Math is
81 `~/stan-dev/math/`:
82 
83 ```bash
84 > make -j4 -f ~/stan-dev/math/make/standalone math-libs
85 > make -f ~/stan-dev/math/make/standalone foo
86 ```
87 The `math-libs` target has to be called only once, and can be omitted
88 for subsequent compilations.
89 
90 The standalone makefile ensures that all the required `-I` include
91 statements are given to the compiler and the necessary libraries are
92 linked: `~/stan-dev/math` and `~/stan-dev/math/lib/eigen_3.3.9` and
93 `~/stan-dev/math/lib/boost_1.75.0` and
94 `~/stan-dev/math/lib/sundials_5.6.1/include` and
95 `~/stan-dev/math/lib/tbb_2019_U8/include`. The
96 `~/stan-dev/math/lib/tbb` directory is created by the `math-libs`
97 makefile target automatically and contains the dynamically loaded
98 Intel TBB library. The flags `-Wl,-rpath,...` instruct the linker to
99 hard-code the path to the dynamically loaded Intel TBB library inside
100 the stan-math directory into the final binary. This way the Intel TBB
101 is found when executing the program.
102 
103 Note for Windows users: On Windows the `-rpath` feature as used by
104 Stan Math to hardcode an absolute path to a dynamically loaded library
105 does not work. On Windows the Intel TBB dynamic library `tbb.dll` is
106 located in the `math/lib/tbb` directory. The user can choose to copy
107 this file to the same directory of the executable or to add the
108 directory `/path/to/math/lib/tbb` as absolute path to the system-wide
109 `PATH` variable.

111 Intel TBB
112 ---------
113 
114 `math` now supports the new interface of Intel TBB and allows using external library (e.g., with [`oneTBB`](https://github.com/oneapi-src/oneTBB) or the system TBB library), using `TBB_LIB` and `TBB_INC` environment variables.
115 
116 To build the development version of `math` with [`oneTBB`](https://github.com/oneapi-src/oneTBB):
117 
118 - Install [`oneTBB`](https://github.com/oneapi-src/oneTBB).
119 
120 For example, installing [`oneTBB`](https://github.com/oneapi-src/oneTBB) on Linux 64-bit (`x86_64`) to `$HOME` directory (change if needed!):
121 ```bash
122 TBB_VERSION="2021.1.1"
123 
124 wget https://github.com/oneapi-src/oneTBB/releases/download/v2021.1.1/oneapi-tbb-$TBB_VERSION-lin.tgz
125 tar zxvf oneapi-tbb-$TBB_VERSION-lin.tgz -C $HOME
126 
127 export TBB="$HOME/oneapi-tbb-$TBB_VERSION"
128 ```
129 
130 - Set the TBB environment variables (specifically: `TBB` for the installation prefix, `TBB_INC` for the directory that includes the header files, and `TBB_LIB` for the libraries directory).
131 
132 For example, installing [`oneTBB`](https://github.com/oneapi-src/oneTBB) on Linux 64-bit (`x86_64`) to `$HOME` directory (change if needed!):
133 ```bash
134 source $TBB/env/vars.sh intel64
135 
136 export TBB_INC="$TBB/include"
137 export TBB_LIB="$TBB/lib/intel64/gcc4.8"
138 ```
139 
140 - Set `Stan` local compiler flags to use the new TBB interface:
141 ```bash
142 mkdir -p ~/.config/stan
143 echo TBB_INTERFACE_NEW=true>> ~/.config/stan/make.local
144 ```
145 
146 Compilers
147 ---------
148 
149 The above example will use the default compiler of the system as
150 determined by `make`. On Linux this is usually `g++`, on MacOS
151 `clang++`, and for Windows this is `g++` if the RTools for Windows are
152 used. There's nothing special about any of these and they can be
153 changed through the `CXX` variable of `make`. The recommended way to
154 set this variable for the Stan Math library is by creating a
155 `make/local` file within the Stan Math library directory. Defining
156 `CXX=g++` in this file will ensure that the GNU C++ compiler is always
157 used, for example. The compiler must be able to fully support C++11
158 and partially the C++14 standard. The `g++` 4.9.3 version part of
159 RTools for Windows currently defines the minimal C++ feature set
160 required by the Stan Math library.
161 
162 Note that whenever the compiler is changed, the user usually must
163 clean and rebuild all binary dependencies with the commands:
164 ```bash
165 > make -f path/to/stan-math/make/standalone math-clean
166 > make -j4 -f path/to/stan-math/make/standalone math-libs
167 ```
168 This ensures that the binary dependencies are created with the new
169 compiler.
